# Linsford Group Website

Static marketing site for Linsford Group (customs clearing & freight forwarding).

## Structure

```
src/
  pages/     Every page AND every dc-import component (site-nav.html,
             site-footer.html, spotlight.html) — see constraint below for why
             components live here instead of a separate folder.
  styles/    main.css (shared reset/typography), responsive.css (all mobile
             breakpoints + shared animations), header.css, footer.css
  js/        support.js, image-slot.js — vendored runtime, generated by an
             external build ("dc-runtime"). Do not hand-edit; there is no
             build step in this repo to regenerate them from source.
  assets/
    images/  photos used across the site
    videos/  cape-town-port.mp4 — homepage hero background video
public/      static files served as-is (robots.txt)
scratch/     builder-tool artifacts kept for reference, not part of the site
             (empty test canvases, a project thumbnail, an old zip backup)
```

There is no `dist/` or `package.json` — nothing here is compiled or bundled.
Every page is plain HTML loaded directly by the browser.

## Important constraint: dc-import requires same-folder siblings

Every page loads its header, footer, and a cursor-glow effect at runtime via:

```html
<dc-import name="site-nav" ...></dc-import>
<dc-import name="site-footer" ...></dc-import>
```

`src/js/support.js` resolves these by fetching `./<name>.html` **relative to
the page that imports it** (`COMPONENT_DIR` is hardcoded to `"."` inside the
generated runtime). That means:

- `site-nav.html`, `site-footer.html`, and `spotlight.html` **must stay in the
  same folder as every page that imports them** (`src/pages/`). Moving them
  into a separate `components/` folder will silently break the header/footer
  on every page — dc-import will fail to fetch them and render nothing, with
  only a console error to show for it.
- `.image-slots.state.json` (image-placeholder tool state) is fetched the
  same way and must also stay next to the pages in `src/pages/`.

Script tags (`support.js`, `image-slot.js`) and asset paths (`images/...`)
are plain attributes, not fetched-by-name — those can point anywhere, which
is why they were safely moved into `src/js/` and `src/assets/` with every
reference rewritten.

## Running locally

This site can't be opened as a `file://` URL — `dc-import`'s `fetch()` calls
are blocked by CORS under the file protocol, so the header/footer won't load
no matter the screen size. Serve it over HTTP instead, e.g.:

```
npx http-server . -p 8080
```

then open `http://localhost:8080/src/pages/index.html`.

## Known gaps

- Downloads page links and the footer's Terms/Disclaimer/Privacy links are
  all placeholder `href="#"`.
- The site-nav "active" state doesn't recognize `quote` as a page key, so no
  nav link highlights when viewing the quote page (by design — Quote isn't
  one of the primary nav items, it's the standalone CTA button).
